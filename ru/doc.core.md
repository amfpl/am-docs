# Как начать

<i id="download"></i>
## Установка

<i id="dl-win"></i>
### Установка для Windows
1. Перейдите на [GitHub Releases для Am](https://github.com/amfpl/am/releases)
2. Найдите нужную<b><sup>\*</sup></b> вам версию
3. Установите `amc-*_setup.exe`
4. Запустите установленный файл
5. Выберите путь установки
6. Готово!

> [!IMPORTANT]
> <b><sup>\*</sup></b> Если вы не знаете какую версию установить, то рекомендуем последнюю версию с пометкой `stable`

<i id="dl-linix"></i>
### Установка для Linux
1. Перейдите на [GitHub Releases для Am](https://github.com/amfpl/am/releases)
2. Найдите нужную<b><sup>\*</sup></b> вам версию
3. Установите `amc-*_setup`
4. Запустите установленный файл
5. Выберите путь установки
6. Готово!

> [!IMPORTANT]
> <b><sup>\*</sup></b> Если вы не знаете какую версию установить, то рекомендуем последнюю версию с пометкой `stable`

<i id="check-for-install"></i>
## Проверка
Чтобы узнать, успешно ли установился компилятор вы можете ввести следущие команды:
- **Поверхностная проверка**. `am --version` должно вывести: `Am Compiler v0.1.0` (на месте `v0.1.0` может быть любая другая версия)
- **Глубокая проверка**. `am deep` должно вывести `Success` (если вы использовали `am deep --verbose`, то `Success` ищите на первой строке, дальше идет отладочная информация о каждом модуле)

<i id="first-start"></i>
## Первый запуск
Как и в любом другом языке, первым проектом должен быть `Hello, world!`.

Создайте файл с расширением `*.upc` и следующим содержанием:
```upc
#[main]
  version="0.1.0"
  name="hello_world"

#[build]
  main={./main.am}
  platform=@["win"] #!: или "linux"
  test=[:test_build]
  release=[:release_build]

#[test_build]
  outfile=("test_" + name + "-" + version)
  outpath={./testificate/}

#[release_build]
  outfile=(name + "-" + version)
  outpath={./bin/}
```

Создайте файл `main.am` в той же папке, где и `*.upc` со следующим содержанием:
```java
import.addSpace("std/io", io, io());
program.addSpace("hello", hi() {
  io.outputln("Hello, world!");
  program.end(0);
})(use io);
```

После выполните в консоли:
```bash
cd путь/до/созданных/файлов
am build
# Если am не распознан, как имя командлета добавьте папку bin, где лежит am, в PATH, или используйте прямой путь:
path/to/am.exe build
# или, если у вас Linux:
path/to/am build
```

Когда компиляция прошла - запустите сгенерированный файл, он должен показать `Hello, world!`.

<i id="cli"></i>
# CLI
CLI (или же Command-Line Interface) - это взаимодействие пользователя с программой через консоль.

Для взаимодействия Am с пользователем через консоль используется команда `am <args>`

При неверном значении аргумента или команды возникает следущая ситуация:
```bash
Value of "..." indicated incorrectly.
Do you want to continue ignoring this value? [Y/n]: 
```

<i id="check-cmds"></i>
## Проверочные команды
Команды, которые только проверяют, не изменяя / не выполняя побочные действия:
- **`am --help`** или любая неверная команда выводит сообщение со списком всех команд и их объяснением
- **`am --version`** или **`am -v`** выводит номер установленной версии
- **`am --docs [lang]`** по умолчанию (если `lang` пустое) выводит ссылку на английскую версию документации, если вместо `lang` вставить конкретное значение (`ru`, `en`), то ссылка будет ввести на документацию с тем языком, который был указан на месте `lang`
- **`am --current-profile`** выводит выбранный профиль (какой C компилятор используется, по умолчанию: `auto` - встроенный TCC)
- **`am --profile-list`** выводит все профили и полные пути к их исполняемым файлам
- **`am --explain <code>`** объяснить ошибку по её коду `Error+xxxx` или более краткой форме `e:xxxx`
- **`am --err-codes-list`** выводит список всех кодов ошибок и краткого их описания

<i id="executable-cmds"></i>
## Исполняемые команды
Команды который генерируют / меняют / выполняют что-то:
- **`am build`** проводит релизную сборку, собирает проект в папку, указанную в конфигурации (по умолчанию, в папку `bin`)
- **`am test`** проводит тестовую сборку, собирает проект в папку, указанную в конфигурации (по умолчанию, в папку `testificate`)
- **`am add "name" : "path/to/executable"`** добавляет свой профиль, который в последствии можно выбрать
- **`am set "name"`** выбирает профиль, узнать список всех профилей можно с помощью `am --profile-list`
- **`am delete "name" !confirm`** удаляет профиль<b><sup>\*</sup></b> (`!confirm` обязателен, если его не будет ничего не удалится)
- **`am deep`** создает проект в кэше, компилирует его для всех операционных систем, проверяя тем самым почти все функции компилятора на их работоспособность (компилируемый код содержит вызовы всех функций и конструкций из ядра и стандартной библиотеки), после проверки проект удаляется из кэша
- **`am clean [level]`** если `level` будет `0` удаляет весь кэш, `testificate` и `bin` проекта (или другие папки, в зависимости от выбранных для сборки в конфигурации), если `level` будет `1` удаляет только кэш проекта, если `level` будет `2` удаляет только `testificate` и `bin` (или другие папки, в зависимости от выбранных для сборки в конфигурации)<b><sup>\*\*</sup></b>

> [!IMPORTANT]
> <b><sup>\*</sup></b> Профиль `auto` удалить **невозможно**

> [!IMPORTANT]
> <b><sup>\*\*</sup></b> По умолчанию `level` равен `2`

<i id="args"></i>
## Аргументы
Аргументы нужны для более удобного использования команд, они пишутся в самом конце строки:
- **`--ast`** или **`-a`** после команды `build` или `test` выводит AST структуру узлов, если использовано с другими командами - игнорируется
- **`--tokens`** или **`-t`** после команды `build` или `test` выводит список токенов, если использовано с другими командами - игнорируется
- **`--verbose`** или **`-d`** выводит более подробную информацию, игнорируется для `am --version`, `am --help`, `am --docs`, `am add`, `am set`, `am delete` и `am --explain`
- **`--save-c`** или **`-c`** после команды `build` или `test` сохранит сгенерированные C файлы, если использовано с другими командами - игнорируется
- **`--save-logs=[path]`** или **`-l=[path]`** сохраняет все логи в указанный path в формате `log_date_time.txt`, `path` указывается от корня текущего проекта (от файла `*.upc`)

<i id="datatypes"></i>
# Типы данных
Типы данных - это просто разделение значений на виды, что бы сделать их удобнее и безопаснее. Каждый тип может хранить какое-то значение, которое не может хранить другой тип.

<i id="type-string"></i>
## Строки

Тип данных строка (**`string`**) может хранить текст, который можно в последствии вывести в консоль, на экран или использовать внутри программы.

Строки обязательно обернуты двойные кавычки (`"`: ASCII-номер 34).

Если сложить 2 строки с помощью оператора сложения (`+`), то на выходе получится тип `string`.

Тип `string` нельзя складывать, вычитать, делить, умножать или возводить в степень с другими типами, кроме `int` и `char`.

Тип `int` **можно** умножать на `string`, тогда значение строки будет повторяться столько раз, сколько написано в значении `int` (это работает только с теми `int`, который больше нуля), остальные операции для `int` вместе со строками недоступны. [См. пример 1](#type-string-example-1).

Тип `char` **можно** умножать на тип `string`. В связи с тем, что тип `char` по своей сути - это и `string`, и `int` одновременно, то значение будет повторяться столько раз, сколько получится в значении при переводе `char` в `int` по ASCII таблице символов (например, буква `a` в ASCII это номер `97`, значит значение повторится 97 раз). [См. пример 2](#type-string-example-2).

Тип `char` **можно** складывать с типом `string`, на выходе получится строка. [См. пример 3](#type-string-example-3).

### Примеры

<i id="type-string-example-1"></i>
Пример 1
```java
1 * "hello " // = "hello "
2 * "hello " // = "hello hello "
3 * "hello " // = "hello hello hello "
```

<i id="type-string-example-2"></i>
Пример 2
```java
'•' * "hello "
/*
'•' при переводе в int даст 7
7 * "hello " даст "hello hello hello hello hello hello hello "
*/
```

<i id="type-string-example-3"></i>
Пример 3
```java
'h' + "ello" // = "hello"
```

<i id="type-char"></i>
## Символы

Тип данных символ (**`char`**) может хранить 1 символ *или* escape-последовательность. Хранить 2 и более символа или нескольно последовательностей **нельзя**.

В данный момент `char` может хранить только ASCII-символы, UTF-8 не доступно (будет ли когда-нибудь - неизвестно).

Так как `char` одновременно и строка, и число, то взаимодействовать он тоже может со строками и числами.

Взаимодействие со строками вы можете увидеть в [предыдущем разделе](#type-string).

Взаимодействие с числами разделяется на 2 группы: целыми и дробными. Если `char` взаимодействует с дробными числами (`float`, `double`), то выходное значение (без сторонних вмешательств) всегда будет дробным, а если с целыми, то выходное значение может быть как целым, так и дробным.

Тип `char` с `int` может умножаться, делиться, возводиться в степень, вычитаться и складываться. Это происходит, потому что `char` хранит данные о номере ASCII-символа, и, как следствие, может стать числом. [См. пример 1](#type-char-example-1).

Тип `char` с `float` (и `double`) может так же умножаться, делиться, возводиться в степень, вычитаться и складываться. [См. пример 2](#type-char-example-2).

Тип `char` так же может хранить escape-последовательности. Вот некоторые из них: `\n` (новая строка - создает новую строку), `\t` (табуляция - ставит 4 пробела), `\\` (обратный слэш - заменяет на `\` - такое действие называется экранирование<b><sup>\*</sup></b>), `\'` (одна кавычка - получить `'`). [См. пример 3](#type-char-example-3).

> [!NOTE]
> <b><sup>\*</sup></b> Экранирование - это игнорирование специфичных символов. Например, если требуется вывести в консоль `"Hello!"`, то нужно прибегнуть к экранированию `"\"Hello!\""`, потому что простое ""Hello!"" вызвало бы ошибку (`""` - первая строка, `Hello!` неизвестное значение, `""` - вторая строка)

### Примеры

<i id="type-char-example-1"></i>
Пример 1
```java
8   *  'd' // = 8   *  100 = 800
1   /  'd' // = 1   /  100 = 0.01
200 /  'd' // = 200 /  100 = 2
101 -  'd' // = 101 -  100 = 1
93  -  'd' // = 93  -  100 = -7
8   +  'd' // = 8   +  100 = 108
'd' ** 2   // = 100 ** 2   = 10000
'd' >  50  // true
'd' >  100 // false
```

<i id="type-char-example-2"></i>
Пример 2
```java
7.5   *  'd'     // = 7.5   *  100 = 750
150.5 /  'd'     // = 150.5 /  100 = 1,505
101.5 -  'd'     // = 101.5 -  100 = 1.5
10.5  -  'd'     // = 10.5  -  100 = -89.5
5.5   +  'd'     // = 5.5   +  100 = 105.5
'd'   ** 2.5     // = 100   ** 2.5 = 100000
'd'   >  99.99   // true
'd'   >  100.00  // false
```

<i id="type-char-example-3"></i>
Пример 3
```java
'\'' // = '
'\n' // =
     // перенос строки
'\\' // = \
'\t' // =     
     //   ^^^^
     //   ||||
     //   1234
```

<i id="types-number"></i>
## Числа

Числа делятся на 2 категории: целые (**`int`**) и дробные (**`float`** и **`double`**).

<i id="type-int"></i>
## Целые числа

Целые числа хранят только числа без запятой.

> [!WARNING]
> Числа в Am не делятся на знаковые и без знаковые. И `-21` и `64` будет целым `int`.

Взаимодействовать целые числа могут с любыми другими числами, [со строками](#type-string) и [с символами](#type-char). [См. пример 1](#type-int-example-1)

### Примеры

<i id="type-int-example-1"></i>
Пример 1
```java
8 == 8 // true
8 >  8 // false
8 *  8 // = 64
8 /  8 // = 1
8 -  8 // = 0
8 +  8 // = 16
8 ** 8 // = 64
```

<i id="type-float-double"></i>
## Дробные числа

Дробные числа хранят только числа с запятой (вместо запятой используется `.`).

Взаимодействовать целые числа могут с любыми другими числами, [со строками](#type-string) и [с символами](#type-char). [См. пример 1](#type-float-double-example-1).

Тип `double` имеет большую длину и точность, чем `float`. К примеру: `float` может хранить только `1.2854805` - это **7** знаков после запятой<b><sup>\*</sup></b> *(цифры дальше - неточные, они могут быть как просто нулями `1.2854805000`, так и мусорными ошибками, поэтому Am не поддерживает такие длинные цифры, округляя их автоматически до `1.2854805`)*, а `double` может хранить уже `1.285480508995335` - это **15** знаков после запятой<b><sup>\*</sup></b> *(цифры дальше - неточные, они могут быть как просто нулями `1.28548050899533500000`, так и мусорными ошибками, поэтому Am не поддерживает такие длинные цифры, округляя их автоматически до `1.285480508995335`)*

> [!IMPORTANT]
> <b><sup>\*</sup></b> При условии, что до запятой используется только 1 цифра

### Примеры

<i id="type-float-double-example-1"></i>
Пример 1
```java
8.5 == 8   // false
8.5 >  8   // true
8.5 *  8   // = 68
8.5 /  8   // = 1,0625
8.5 -  8   // = 0.5
8.5 +  8   // = 16.5
8.5 ** 0.8 // = 6.8
```

> [!WARNING]
> Не стоит использовать везде где только можно `float` - это способсвует меньшей оптимизации, т.к. на вычисление `float` требуется больше ресурсов.

<i id="type-bool"></i>
## Булевые значения

Тип булева (**`bool`**) может хранить только 2 значения `true` (правда) и `false` (не правда). Тип нужен в основном только для работы конструкций.

При конвертации в числа `true` превращается в `1` или `1.00`, а `false` в `0` или `0.00`.

Так же при конвертации числе в булева все четные числа `false` (..., `-2`, `0`, `2`, ...), а все нечетные в `true` (..., `-3`, `-1`, `1`, `3`, ...).

<i id="type-array"></i>
## Массивы

Массив (**`arr`**) это список значений, заключенных внутри квадратных скобок (`[` и `]`). Внутри могут быть значения разных типов, перечисленных через запятую. [См. пример 1](#type-array-example-1).

Так как это просто перечисление значений, то и получить значение по ID не получится. Получить значение можно через индекс - номер этого значения в списке. Так как каждый элемент отделен запятой, то можно понять, что индекс можно посчитать по формуле: `номер_элемента - 1`. Отнять 1 нужно, потому что отсчет начинается с `0`. То есть `[0, 1, 2, 3]`.

Вложенные массивы считаются одним значением `[1, true, [0, 7], false]` - в этом массиве всего 4 элемента первый `1` (индекс `0`), второй `true` (индекс `1`), третий `[0, 7]` (индекс `2`), четвертый `false` (индекс `3`).

Комментарии не являются элементом, поэтому ставить запятую после не нужно. [См. пример 2](#type-array-example-2).

### Примеры

<i id="type-array-example-1"></i>
Пример 1
```java
[3.14, true, 51, -2.71, -15, ["hello", 'c'], false, "hi", {obj: "in"}]
```

<i id="type-array-example-2"></i>
Пример 2
```java
[3.14, true, /* Комментарий */ "hi"]
[3.14, true /* Я не поставил запятую после true, поэтому она тут --> */, "hi"]
```

<i id="type-dict"></i>
## Словари

Словари (**`dict`**) это объект, который тоже имеет статическую типизацию. Общение со словарями требует так же знания месте, куда он записан. Об этом будет рассказано позже - [См. глава "Переменные"](#variables).

Значение словаря заключено в фигурные скобки (`{` и `}`) и выглядит как JSoN-подобный синтаксис, только ключи не имеют кавычек: `{name<S>: "John", age<I>: 15}`.

Названия (ключи) должны иметь в имени только `[A-Za-z0-9_]`.

Так как это статический тип, то типы оббявляются заранее и указаны рядом с ключем в `< >`. Типы для массивов имеют альтернативные именования, написав `{age<int>: 15}` вы увидите ошибку. Внизу список как выглядят типы:

| Тип данных        | Указание в словаре |
| ----------------- | ------------------ |
| `string`          | `S`                |
| `char`            | `C`                |
| `int`             | `I`                |
| `float`           | `F`                |
| `double`          | `D`                |
| `arr`             | `A`                |
| `dict`            | `T`                |
| `void`            | `V`                |
| `auto`            | `U`                |

<i id="type-void"></i>
## Пустоты

Тип данных пустоты (**`void`**) может хранить только специальное значение `null`. Он нужен для определения констант посреди кода. Чтобы передача `null` значения тоже типизировалось.

Так же `void` может быть совместим с другими элементарными типами - `string`, `char`, `int`, `float`, `double`, тогда эти типы будут иметь приставку `voidable_`. Это нужно, что бы типы могли иметь `null`, но при этом сохранять статическую типизацию: в `voidable_int`, например, нельзя присвоить строку `"hello"`.

Типы `voidable_*` не нужно объявлять отдельно. Они появлявляются только если элементарный тип имеет значение `null`. А потом пропадают, как только значение снова соответствует типу.

<i id="type-default"></i>
## Автоматический тип

Автоматический тип не имеет своего индивидуального знаения. Он может принимать любое значение, но только первый раз. Как только значение объявлено, `default` определяет его тип и становится этим типом.

Например, в тип `default` передали `"Hello"` - теперь `default` это `string`, а значит больше не может получить значение другого типа. Он просто определяет тип автоматически, но не является динамическим.

<i id="variables"></i>
# Переменные

Переменные это способ хранить данные и менять их.

<i id="var-declar"></i>
## Объявления

Переменные объявляются с помощью `M[T] N = V` где:
- `M` это модификатор: `declar` или `const`<b><sup>\*\*</sup></b>
- `T` это тип переменной
- `N` это ее имя<b><sup>\*</sup></b>
- `V` это ее значение

> [!IMPORTANT]
> <b><sup>\*</sup></b> Имя может содержать только `[A-Za-z0-9_]`.<br>
> <b><sup>\*</sup></b> Имя не должно содержать пробелов.

> [!IMPORTANT]
> <b><sup>\*\*</sup></b> Если используется `const`, то переменная фиксированная, ее нельзя изменить.<br>
> <b><sup>\*\*</sup></b> Если используется `declar`, то переменная изменяемая.

<i id="var-use"></i>
## Использование

### Замена

Тут имя переменной просто заменяется на объявленное значение

Использовать переменную можно через `N`, где:
- `N` это объявленной имя переменной

Переменные можно вставить в почти любую часть, где ожидается значение:
- **Вызов функции**. `F(N)`, где:
  - `F` это объявленной имя функции
  - `N` это объявленной имя переменной
- **Передача переменным**. `declar[T] V = N`, где:
  - `T` это тип новой переменной
  - `V` это имя новой переменной
  - `N` это имя объявленной переменной
- **Сравнения**. `(N > 10) ? continue : break`, где:
  - `N` это объявленной имя переменной
- **Арифметика**. [См. пример 1](#type-var-use-example-1).
- **Циклы**. [См. пример 2](#type-var-use-example-2).
- *многие другие...*

### Массивы

Получить массив полностью можно просто с помощью `N`, где:
- `N` это имя переменной куда записан массив

Получить значение из массива можно с помощью `N[I]`, где:
- `N` это имя переменной, куда записан массив
- `I` это индекс элемента (`номер элемента - 1` - [См. Тип данных Массив](#type-array))

Получить значение вложенного массива можно с помощью двух переменных ([См. Пример 3]()) или через двойной индекс ([См. Пример 4]()) `N[I][S]`, где:
- `N` это имя контейнера с родительным массивом
- `I` это индекс элемента вложенного массива внутри `N`
- `S` это нужный элемент из вложенного массива

### Словари

Получить словарь полностью можно с помощью `N`, где:
- `N` это имя контейнера куда объявлен словарь

Если нужно получить значение из словаря (См. Пример 5), то можно использовать `N.K`, где:
- `N` это имя контейнера, где объявлен словарь
- `K` это имя ключа в словаре

При чем, если в ключе находится массив (См. Пример 6) вы можете сразу получить нужное значение с помощью `N.K[I]`, где:
- `N` это имя контейнера, где объявлен словарь
- `K` это имя ключа в словаре, куда объявлен массив
- `I` индекс элемента внутри массива, объявленного в словаре

Так же можно делать и с вложенными массивами (См. Пример 7): `N.K[I][S]`, где:
- `N` это имя контейнера, где объявлен словарь
- `K` это имя ключа в словаре, куда объявлен массив
- `I` индекс вложенного массива внутри массива из ключа
- `S` это нужный элемент из вложенного массива

### Примеры

<i id="type-var-use-example-1"></i>
Пример 1
```java
declar[int] a = 10;
declar[int] b = 20;
declar[int] c = 2;
c = a + b * c; // = 10 + 20 * 2 = 10 + 40 = 50
```

<i id="type-var-use-example-2"></i>
Пример 2
```java
declar[int] i = 0;
while (i < 10) {
  // тут что-то может быть
  i++;
}
```

<i id="var-redeclar"></i>
## Переназначение

Переназначение позволяет поменять значение переменной на значение того же типа, что и было предыдущее с помощью `N = V`, где:
- `N` это имя **уже сущесвующей** переменной<b><sup>\*</sup></b>
- `V` это новое значение переменной

> [!WARNING]
> <b><sup>\*</sup></b> Работает только если `N` существовала раньше и не была удалена досрочно.

При этом, пока процесс переназначения не закончился можно использовать старое значение. [См. пример 1](#type-var-redeclar-example-1).

### Примеры

<i id="type-var-redeclar-example-1"></i>
Пример 1
```java
declar[int] x = 90;
x = x + 10; // = старое x (т.к. после выражения ; еще не было) + 10 = 90 + 10 = 100
```

<i id="var-forward-declaration"></i>
## Ранняя декларация

Позволяет объявлять переменную, не добавляя конкретное значение во время декларации. Значение автоматически станет `null`.

Объявляется как `declar[T] N`, где:
- `T` это тип переменной
- `N` это имя переменной<b><sup>\*</sup></b>

> [!IMPORTANT]
> <b><sup>\*</sup></b> Имя может содержать только `[A-Za-z0-9_]`.<br>
> <b><sup>\*</sup></b> Имя не должно содержать пробелов.

Позже, через переназначение можно добавить значение. [См. пример 1](#type-var-forward-declaration-example-1).

### Примеры

<i id="type-var-forward-declaration-example-1"></i>
Пример 1
```java
declar[int] x;
x = 90;
```

<i id="var-shadowing"></i>
## Затенение

Затенение (shadowing) это объявление новой переменной, поверх текущей с тем же именем. [См. пример 1](#type-var-forward-declaration-example-1). При выходе из области видимости, где была создана теневая переменная, она изчезает и возвращает старое значение, если таковое было. [См. пример 2](#type-var-forward-declaration-example-2), [См. пример 3](#type-var-forward-declaration-example-3).

### Примеры

<i id="type-var-forward-declaration-example-1"></i>
Пример 1
```java
declar[int] x = 90;
// x = 90
declar[string] x = "hello";
// x = "hello"
```

<i id="type-var-forward-declaration-example-2"></i>
Пример 2
```java
declar[int] x = 90;
// x = 90
{
  declar[string] x = "hello";
  // x = "hello"
}
// x = 90
```

<i id="type-var-forward-declaration-example-3"></i>
Пример 3
```java
{
  declar[int] x = 90;
  // x = 90
  declar[string] x = "hello";
  // x = "hello"
}
// x не найден
```

<i id="var-retype"></i>
## Смена типа

Смена типа происходит с помощью оператора `->`. При чем старое значение безвозратно удаляется.

Конструкция смены типа выглядит как `N->T = V`, где:
- `N` это имя **уже сущесвующей** переменной<b><sup>\*</sup></b>
- `T` это новый тип переменной
- `V` это новое значение переменной<b><sup>\*\*</sup></b>

> [!WARNING]
> <b><sup>\*</sup></b> Работает только если `N` существовала раньше и не была удалена досрочно.

> [!WARNING]
> <b><sup>\*\*</sup></b> Если значение не указано (как в [ранней декларации](#var-forward-declaration)), то значение автоматически `null`.

### Примеры

<i id="type-var-retype-example-1"></i>
Пример 1
```java
declar[int] x = 90;
x->float = x + 0.5;
```

<i id="var-clear"></i>
## Очистка

Хотя память освобождается автоматически благодаря GC (Garbage Collecter), если вам все таки пригодится очистить вы можете вызвать метод `down()` или `downl()`.

`down()` удаляет переменную полностью. [См. пример 1](#type-var-clear-example-1).

`downl()` удаляет лишь последнюю оболочку, созданную с помощью затенения. [См. пример 2](#type-var-clear-example-2). (если до этого затенение не приминялось или было не найдено, то переменная удалится полностью, как и в случае с `down()`. [См. пример 3](#type-var-clear-example-3)).

### Примеры

<i id="type-var-clear-example-1"></i>
Пример 1
```java
declar[int] x = 90;
down(x);
// тут переменная x уже не сущесвует
```

<i id="type-var-clear-example-2"></i>
Пример 2
```java
declar[int] x = 90;
declar[int] x = "hello";
downl(x); // удаляет последний слой - "hello"
// тут переменная x равна 90
```

<i id="type-var-clear-example-3"></i>
Пример 3
```java
declar[int] x = 90;
downl(x); // удаляет последний слой - 90, больше слоев нет
// тут переменная x уже не сущесвует
```

# Операторы
Операторы - это символ или группа символов, которые позволяют связывать компоненты (операнды) между собой и выполнять действия.

Операторы бывают буквенные - состоящие из букв (`AND`, `OR`, `NOT`), и, соответственно, символьные - состоящие из символов (`>`, `&&`, `==`)

Операторы делятся на 4 категории по их назначению:
- [Математические (арифметические)](#ops-math)
- [Сравнения](#ops-compare)
- [Логические](#ops-logical)
- [Битовые](#ops-bitwise)
- [Назначения](#ops-assign)
- [Специальные](#ops-special)
- [Потоковые](#ops-stream)

Операторы делятся на 3 категории по количеству операндов:
- [Унарные](#ops-unary) (От лат. unarius - одинарный)
- [Бинарные](#ops-binary) (От лат. binarius - двойной, оператор имеет 2 операнда)
- [Тернарные](#ops-ternary) (От лат. ternarius - тройной, оператор имеет 3 операнда)

<i id="as-intend"></i>
## Категория: По назначению

<i id="ops-math"></i>
### Математические

Математические операторы нужны для математических вычислений.
Узнать приоритеты операторов (какие выполняются первыми) можно в общей таблице операторов.

К математическим операторам относятся:
- **`+`** Сложение
  - При использовании между операндами `int` идет арифметическое сложение (`1 + 1 = 2`), тип выходного значения `int`
  - При использовании между операндами `float` идет арифметическое сложение (`1.75 + 1.25 = 3`), тип выходного значения `float`
  - При использовании между операндами `double` идет арифметическое сложение (`2.9997 + 0.0003 = 3`), тип выходного значения `double`
  - При использовании между операндами `string` идет конкатенация (`"hi, " + "friend" = "hi, friend"`), тип выходного значения `string`
  - При использовании между операндами `char` идет конкатенация (`'c' + 'a' = "ca"`), тип выходного значения `string`
  - При использовании между операндами `bool` идет логическое `ИЛИ` (`true + true = true`, `true + false = true`, `false + false = false`), тип выходного значения `bool`
  - При использовании между операндами `arr` идет объединение множеств (`[3.14, "hello"] + [3.14, "ye"] = [3.14, "hello", 3.14, "ye"]` - элементы первого массива идут **ДО** элементов второго, порядок элементов каждого массива не изменен), тип выходного значения `arr`
  - При использовании между операндами `dict` идет объединение ключей (`{hi<S>: "Hello"} + {pi<F>: 3.14, hi<S>: "Hi"} = {pi<F>: 3.14, hi<S>: "Hi"}` - если ключи повторяются, то ключ последнего словаря заменяет ключ первого), тип выходного значения `dict`
  - При использовании между операндами `void` значение остается без изменений (`null + null = null`), тип выходного значения `void` или `voidable_*` (`voidable_*` типы смешивать нельзя)
  - Взаимодействие типов:
    - `int` + `int` -> `int`
    - `int` + `float` -> `float`
    - `int` + `double` -> `double`
    - `int` + `string` -> ERROR
    - `int` + `bool` -> `int` (`true` = `1`, `false` = `0`)
    - `bool` + `int` -> `bool`
      - `true` + `int` (положительный) -> `true`
      - `true` + `int` (отрицательный) -> `false`
      - `true` + `int` (`0` или `-0`) -> `true`
      - `false` + `int` (положительный) -> `true`
      - `false` + `int` (отрицательный) -> `false`
      - `false` + `int` (`0` или `-0`) -> `false`
    - `int` + `char` -> `int`
    - `char` + `int` -> `char` или `int` (если результат `int` (`char`) + `int` соответствует значению какого-нибудь символа из ASCII таблицы, то выходной тип - `char`, иначе - `int`)
    - `int` + `arr` -> `arr` (значение `int` просто добавляется в конец массива)
    - `int` + `dict` -> ERROR
    - `int` + `void` -> `int` (без изменений)
    - `float` + `float` -> `float`
    - `float` + `double` -> `double`
    - `float` + `string` -> ERROR
    - `float` + `char` -> `float`
    - `char` + `float` -> `char` / `float` (если результат `int` (`char`) + `float` (округленного до `int` по математическим правилам) соответствует значению какого-нибудь символа из ASCII таблицы, то выходной тип - `char`, иначе - `float`)
    - `float` + `bool` -> `float` (если у `bool` значение `true`, то `float` + `1` (`1.00`), а если `false`, то `float` + `0` (`0.00`))
    - `bool` + `float` -> `bool` (в этом случае `float` округляется по математическим правилам до `int`)
      - `true` + `float` (положительный `int`) -> `true`
      - `true` + `float` (отрицательный `int`) -> `false`
      - `true` + `float` (`0` или `-0`) -> `true`
      - `false` + `float` (положительный `int`) -> `true`
      - `false` + `float` (отрицательный `int`) -> `false`
      - `false` + `float` (`0` или `-0`) -> `false`
    - `float` + `arr` -> `arr` (значение `float` просто добавляется в конец массива)
    - `float` + `dict` -> ERROR
    - `float` + `void` -> `float` (без изменений)
    - `double` + `string` -> ERROR
    - В остальном `double` работает так же, как `float`, но с большей точностью
    - `string` + `char` -> `string`
    - `string` + `bool` -> ERROR
    - `string` + `arr` -> `arr` (значение `string` просто добавляется в конец массива)
    - `string` + `dict` -> ERROR
    - `string` + `void` -> `string` (без изменений)
    - `char` + `bool` -> `char` / `int`
    - `bool` + `char` -> `bool` (на выходе всегда получается `true`, потому что `char` не имеет отрицательных значений, исключением является символ со значением `0` - это единственный символ, дающий выходной `false` в любом случае)
    - `char` + `arr` -> `arr` (значение `char` просто добавляется в конец массива)
    - `char` + `dict` -> ERROR
    - `char` + `void` -> `char` (без изменений)
    - `bool` + `arr` -> `arr` (значение `bool` просто добавляется в конец массива)
    - `bool` + `dict` -> ERROR

    - `bool` / `char` -> `bool` (в этом случае `char` округляется по математическим правилам до `int`)
      - `true` / `char` (положительный `int`) -> `false`
      - `true` / `char` (`0`) -> `true`
      - `false` / `char` (положительный `int`) -> `false`
      - `false` / `char` (`0`) -> `true`
    - `char` / `arr` -> ERROR
    - `char` / `dict` -> ERROR
    - `char` / `void` -> `char` (без изменений)
    - `bool` / `arr` -> ERROR
    - `bool` / `dict` -> ERROR
    - `bool` / `void` -> `bool` (без изменений)
    - `arr` / `void` -> `arr` (без изменений)
    - `dict` / `arr` -> ERROR
    - `dict` / `void` -> `dict` (без изменений)
    - Обратите внимание, разные `voidable_*` типы со значением `null` нельзя делить между собой!
- **`**`** Возведение в степень
  - При использовании между операндами `int` идет арифметическое возведение (`10 ** 2 = 100`), тип выходного значения `int`
  - При использовании между операндами `float` идет арифметическое возведение (`7.5 ** 2.1 = 68.806371`), тип выходного значения `float`
  - При использовании между операндами `double` идет арифметическое возведение (`6.94 ** 1.5 = 18.282652542779453`), тип выходного значения `double`
  - При использовании между операндами `string` возникает ошибка
  - При использовании между операндами `char` идет арифметическое возведение (`'c' ** 'a' = INFINITY`), тип выходного значения `char` или `int`,`float`,`double` (если значение `int` (`char`) ** `int` (`int`, `float`, `double`) слишком велико или слишком мало относительно значений крайних символов ASCII таблицы символов, то тип выходного значения будет `int` (или `float`, `double` - в зависимости от количества знаков после запятой (при условии, что до запятой всего 1 цифра): <7 - итоговое значение `float`, иначе `double`), иначе `char`)
  - При использовании между операндами `bool` идет логическое `XOR` - `(a ИЛИ b) И (НЕ a ИЛИ НЕ b)` (`true ** true = false`, `true ** false = true`, `false ** true = true`, `false ** false = false`), тип выходного значения `bool`
  - При использовании между операндами `arr` идет вычитание множеств (`[3.14, "hello"] ** [3.14, "ye"] = ["ye", 3.14, "hello", 3.14]` - сначала множества просто объединяются, затем конечный массив разворачивается), тип выходного значения `arr`
  - При использовании между операндами `dict` возникает ошибка
  - При использовании между операндами `void` значение остается без изменений (`null ** null = null`), тип выходного значения `void` или `voidable_*` (`voidable_*` типы смешивать нельзя)
  - Взаимодействие типов:
    - `int` ** `int` -> `int`
    - `int` ** `float` -> `float`
    - `int` ** `double` -> `double`
    - `int` ** `string` -> ERROR
    - `int` ** `bool` -> `int` (при `bool`, равном `true`, итоговое значение - `int` без изменений, если `bool` равняется `false`, то итоговое значение `int` равен `0`)
    - `bool` ** `int` -> `bool`
      - `true` ** `int` (положительный) -> `false`
      - `true` ** `int` (отрицательный) -> `true`
      - `true` ** `int` (`0` или `-0`) -> `true`
      - `false` ** `int` (положительный) -> `true`
      - `false` ** `int` (отрицательный) -> `false`
      - `false` ** `int` (`0` или `-0`) -> `false`
    - `int` ** `char` -> `int`
    - `char` ** `int` -> `char` или `int` (если результат `int` (`char`) ** `int` соответствует значению какого-нибудь символа из ASCII таблицы, то выходной тип - `char`, иначе - `int`)
    - `int` ** `arr` -> ERROR
    - `int` ** `dict` -> ERROR
    - `int` ** `void` -> `int` (без изменений)
    - `float` ** `float` -> `float`
    - `float` ** `double` -> `double`
    - `float` ** `string` -> ERROR
    - `float` ** `char` -> `float`
    - `char` ** `float` -> `char` / `float` (если результат `int` (`char`) ** `float` (округленного до `int` по математическим правилам) соответствует значению какого-нибудь символа из ASCII таблицы, то выходной тип - `char`, иначе - `float`)
    - `float` ** `bool` -> `float` (при `bool`, равном `true`, итоговое значение - `float` без изменений, при `false` - итоговое значение `0`)
    - `bool` ** `float` -> `bool` (в этом случае `float` округляется по математическим правилам до `int`)
      - `true` ** `float` (положительный `int`) -> `false`
      - `true` ** `float` (отрицательный `int`) -> `true`
      - `true` ** `float` (`0` или `-0`) -> `true`
      - `false` ** `float` (положительный `int`) -> `true`
      - `false` ** `float` (отрицательный `int`) -> `false`
      - `false` ** `float` (`0` или `-0`) -> `false`
    - `float` ** `arr` -> ERROR
    - `float` ** `dict` -> ERROR
    - `float` ** `void` -> `float` (без изменений)
    - `double` ** `string` -> ERROR
    - В остальном `double` работает так же, как `float`, но с большей точностью
    - `string` ** `char` -> ERROR
    - `string` ** `bool` -> ERROR
    - `string` ** `arr` -> ERROR
    - `string` ** `dict` -> ERROR
    - `string` ** `void` -> `string` (без изменений)
    - `char` ** `bool` -> `char` / `int` (при `bool`, равном `true`, итоговое значение - `char` / `int` без изменений, при `false` - итоговое значение `0`; если итоговое значение есть в таблице ASCII символов, то возвращаемый тип - `char`, иначе - `int`)
    - `bool` ** `char` -> `bool` (в этом случае `char` округляется по математическим правилам до `int`)
      - `true` ** `char` (положительный `int`) -> `false`
      - `true` ** `char` (`0`) -> `true`
      - `false` ** `char` (положительный `int`) -> `true`
      - `false` ** `char` (`0`) -> `false`
    - `char` ** `arr` -> ERROR
    - `char` ** `dict` -> ERROR
    - `char` ** `void` -> `char` (без изменений)
    - `bool` ** `arr` -> ERROR
    - `bool` ** `dict` -> ERROR
    - `bool` ** `void` -> `bool` (без изменений)
    - `arr` ** `void` -> `arr` (без изменений)
    - `dict` ** `arr` -> ERROR
    - `dict` ** `void` -> `dict` (без изменений)
    - Обратите внимание, разные `voidable_*` типы со значением `null` нельзя возводить между собой!
